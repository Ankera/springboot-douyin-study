#### 三层架构
```markdown
┌─────────────────────────────────────┐
│  @Controller  (控制层)               │  ← 接收请求，返回JSON
│  UserController                     │
└──────────────┬──────────────────────┘
               ▼ 调用
┌─────────────────────────────────────┐
│  @Service     (业务层)               │  ← 处理业务逻辑，事务管理
│  UserService                        │
└──────────────┬──────────────────────┘
               ▼ 调用
┌─────────────────────────────────────┐
│  @Repository  (数据层)               │  ← 查数据库
│  UserMapper / UserDao               │
└─────────────────────────────────────┘
```

#### @ConfigurationProperties vs @Value 对比
```markdown
| 特性           | `@ConfigurationProperties`       | `@Value`           |
| ------------ | -------------------------------- | ------------------ |
| **绑定方式**     | 批量绑定（整个对象）                       | 单个属性               |
| **松散绑定**     | ✅ 支持（如 `secret-id` → `secretId`） | ❌ 严格匹配             |
| **类型转换**     | ✅ 自动（String→int/List/对象）         | ⚠️ 简单类型，复杂类型麻烦     |
| **校验支持**     | ✅ 支持 `@Validated` + `@NotNull`   | ❌ 不支持              |
| **SpEL 表达式** | ❌ 不支持                            | ✅ 支持 `${}` + `#{}` |
| **适用场景**     | 多字段配置类                           | 单个/少量值             |

```
